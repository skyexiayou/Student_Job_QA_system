## 3. MySQL 数据库

当前项目已经配置为 MySQL：

```text
host: 127.0.0.1
port: 3306
user: root
password: 123456
database: rag_job_qa
```

如果需要重新建库，可以在 DataGrip 控制台执行：

```sql
CREATE DATABASE IF NOT EXISTS rag_job_qa
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

如果你想直接在 DataGrip 中建表，执行下面这段 SQL：

```sql
USE rag_job_qa;

CREATE TABLE IF NOT EXISTS users (
    id INT AUTO_INCREMENT PRIMARY KEY COMMENT '用户主键',
    username VARCHAR(64) NOT NULL UNIQUE COMMENT '登录用户名',
    email VARCHAR(128) DEFAULT NULL COMMENT '邮箱，可作为登录凭据',
    phone VARCHAR(32) DEFAULT NULL COMMENT '手机号，可作为登录凭据',
    display_name VARCHAR(64) NOT NULL COMMENT '页面展示昵称',
    password_hash VARCHAR(256) NOT NULL COMMENT 'PBKDF2 加盐密码哈希',
    created_at VARCHAR(32) NOT NULL COMMENT '创建时间',
    INDEX idx_users_email (email),
    INDEX idx_users_phone (phone)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='系统用户表';

CREATE TABLE IF NOT EXISTS auth_tokens (
    token VARCHAR(64) PRIMARY KEY COMMENT '登录令牌',
    user_id INT NOT NULL COMMENT '关联用户 ID',
    created_at VARCHAR(32) NOT NULL COMMENT '令牌创建时间',
    INDEX idx_auth_tokens_user_id (user_id),
    CONSTRAINT fk_auth_tokens_user_id
        FOREIGN KEY (user_id) REFERENCES users(id)
        ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户登录令牌表';
```

如果你的 `users` 表已经建过但缺少邮箱和手机号字段，执行下面这段升级 SQL：

```sql
USE rag_job_qa;

ALTER TABLE users
ADD COLUMN email VARCHAR(128) DEFAULT NULL COMMENT '邮箱，可作为登录凭据' AFTER username;

ALTER TABLE users
ADD COLUMN phone VARCHAR(32) DEFAULT NULL COMMENT '手机号，可作为登录凭据' AFTER email;

CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_phone ON users(phone);
```

检查表是否建好：

```sql
USE rag_job_qa;
SHOW TABLES;
SHOW FULL COLUMNS FROM users;
SHOW FULL COLUMNS FROM auth_tokens;
```

创建用户表和登录令牌表：

```powershell

D:\Anaconda\python.exe -c "from src.rag_job_qa.config import Settings; from src.rag_job_qa.user_store import UserStore; s=Settings.load(); UserStore(s); print('MySQL 表创建完成')"
```

建表后 DataGrip 中刷新 `rag_job_qa`，应能看到：

- `users`
- `auth_tokens`

```text

```

```text

```

例如：

```text
http://192.168.1.23:8000
```

注意：

- 你的电脑不能休眠或关机。
- PyCharm 运行窗口不能停止。
- 你已关闭防火墙会更方便；若仍无法访问，检查校园网是否开启客户端隔离。
- 如果正在使用 VPN，建议关闭 VPN 后再测试。

## 7. Hugging Face 模型下载到项目路径

项目已经把 Hugging Face 缓存固定到：

```text
E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models
```

不会主动放到 C 盘。

注意：不要直接运行其它网上复制的 Hugging Face 下载命令，否则它很可能默认写入 `C:\Users\你的用户名\.cache`。请使用下面命令，先设置环境变量，再下载。

### 方式一：使用项目下载脚本

```powershell
cd "E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system"
$env:HF_HOME="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface"
$env:HUGGINGFACE_HUB_CACHE="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface\hub"
$env:TRANSFORMERS_CACHE="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface\transformers"
$env:HF_HUB_DISABLE_SYMLINKS_WARNING="1"
$env:HF_ENDPOINT="https://hf-mirror.com"
D:\Anaconda\python.exe scripts\download_hf_model.py --model sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
```

### 方式二：手动下载到一个明确文件夹

如果方式一仍不稳定，用下面命令把模型保存到项目目录下的固定文件夹：

```powershell
cd "E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system"
$env:HF_HOME="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface"
$env:HUGGINGFACE_HUB_CACHE="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface\hub"
$env:TRANSFORMERS_CACHE="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface\transformers"
$env:HF_HUB_DISABLE_SYMLINKS_WARNING="1"
$env:HF_ENDPOINT="https://hf-mirror.com"
D:\Anaconda\python.exe scripts\download_hf_model.py --model sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2 --manual-dir "data\models\paraphrase-multilingual-MiniLM-L12-v2"
```

方式二下载成功后，把 `.env` 中的 `EMBEDDING_MODEL` 改成绝对路径：

```text
EMBEDDING_MODEL=E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\paraphrase-multilingual-MiniLM-L12-v2
USE_HF_EMBEDDING=true
HF_LOCAL_FILES_ONLY=true
```

然后重启系统即可。
