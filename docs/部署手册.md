# 部署手册

本文档用于在 Windows + PyCharm + Anaconda 环境中部署和启动大学生求职岗位知识问答系统。命令默认在 PyCharm 底部 Terminal 或 PowerShell 中执行。

## 1. 环境要求

推荐环境：

- Windows 10/11
- Python 3.10 或 3.11
- Anaconda 或 Miniconda
- PyCharm
- MySQL 8.x
- 可选：Neo4j Desktop 或 Neo4j Server
- 可选：可访问通义千问或其他 OpenAI Compatible 大模型接口

当前示例使用：

```text
Python: D:\Anaconda\python.exe
项目目录: E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system
```

如果你的路径不同，请把命令中的路径替换成自己的实际路径。

## 2. 打开项目

用 PyCharm 打开项目目录：

```text
E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system
```

在 `Settings -> Project -> Python Interpreter` 中选择：


## 3. 安装依赖

进入项目目录：



## 4. 配置 MySQL

系统使用 MySQL 保存用户、登录令牌和求职目标。默认配置在 `src/rag_job_qa/config.py` 中：

```text
host: 127.0.0.1
port: 3306
user: root
password: 空
database: rag_job_qa
```

如果你的 MySQL 密码不是空，建议在项目根目录创建 `.env` 文件：

```text
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=你的MySQL密码
MYSQL_DATABASE=rag_job_qa
```

在 DataGrip、Navicat 或 MySQL 命令行中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS rag_job_qa
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

系统启动时会自动创建或补齐以下表：

- `users`：用户表；
- `auth_tokens`：登录令牌表；
- `user_goals`：求职目标表。

也可以手动触发建表检查：

```powershell
cd "E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system"
D:\Anaconda\python.exe -c "from src.rag_job_qa.config import Settings; from src.rag_job_qa.user_store import UserStore; s=Settings.load(); UserStore(s); print('MySQL 表检查完成')"
```

检查表：

```sql
USE rag_job_qa;
SHOW TABLES;
SHOW FULL COLUMNS FROM users;
SHOW FULL COLUMNS FROM auth_tokens;
SHOW FULL COLUMNS FROM user_goals;
```

## 5. 配置大模型 API Key

系统会按以下优先级读取 API Key：

1. `.env` 文件中的环境变量；
2. 项目外部 `API Key` 目录中的 CSV；
3. `data/api_key/` 目录中的 CSV。

推荐方式是在项目根目录创建 `.env`：

```text
QWEN_API_KEY=你的APIKey
QWEN_BASE_URL=你的OpenAI兼容接口地址
QWEN_MODEL=qwen-plus
```

如果使用 CSV，系统会读取字段：

```text
apiKey
openAiCompatible
```

没有 API Key 时，系统的部分大模型回答能力会受影响；知识库索引、页面和部分本地逻辑仍可检查。

## 6. 配置 Hugging Face Embedding

系统默认使用 `sentence-transformers/all-MiniLM-L6-v2`，并且优先从本地缓存加载：

```text
USE_HF_EMBEDDING=true
HF_LOCAL_FILES_ONLY=true
```

如果本地没有模型，系统会自动降级到 hashing embedding，保证课堂演示可运行。若希望提升检索效果，可以下载模型到项目目录。

推荐下载命令：

```powershell
cd "E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system"
$env:HF_HOME="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface"
$env:HUGGINGFACE_HUB_CACHE="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface\hub"
$env:TRANSFORMERS_CACHE="E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system\data\models\huggingface\transformers"
$env:HF_HUB_DISABLE_SYMLINKS_WARNING="1"
$env:HF_ENDPOINT="https://hf-mirror.com"
D:\Anaconda\python.exe scripts\download_hf_model.py --model sentence-transformers/all-MiniLM-L6-v2
```

也可以在 `.env` 中改成中文效果更好的多语言模型：

```text
EMBEDDING_MODEL=sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
USE_HF_EMBEDDING=true
HF_LOCAL_FILES_ONLY=true
```

下载多语言模型：

```powershell
D:\Anaconda\python.exe scripts\download_hf_model.py --model sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
```

## 7. 配置 Neo4j 可选

Neo4j 不是必需组件。未配置 Neo4j 时，系统仍可完成登录、文档上传、知识问答、岗位助手、模拟面试和基础图谱展示。

如果需要启用 Neo4j，在 `.env` 中添加：

```text
NEO4J_URI=bolt://127.0.0.1:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=你的Neo4j密码
NEO4J_DATABASE=neo4j
NEO4J_VECTOR_INDEX=job_chunk_embedding_index
NEO4J_VECTOR_DIMENSION=384
```

注意：

- Neo4j 服务必须已启动；
- 账号密码必须正确；
- Embedding 维度需要与模型一致，当前默认维度是 384；
- 如果切换 Embedding 模型，需要重建知识库索引。

启动后可访问页面中的知识图谱，也可以打开接口诊断：

```text
http://127.0.0.1:8000/api/graph/diagnostics
```

## 9. 校园网共享访问

如果需要让同一校园网内的伙伴访问，启动时监听所有网卡：

```powershell
cd "E:\26大三下\26课内学习\生产实习\project\rag_student_job_qa_system"
D:\Anaconda\python.exe run.py --host 0.0.0.0 --port 8000
```

查看本机 IPv4 地址：

```powershell
ipconfig
```

找到当前无线网卡或以太网卡下的 `IPv4 地址`，例如：

```text
192.168.1.23
```

伙伴访问：

```text
http://192.168.1.23:8000
```

注意事项：

- 你的电脑不能关机或休眠；
- 运行 `run.py` 的终端不能关闭；
- Windows 防火墙需要允许 Python 或 8000 端口访问；
- 如果校园网开启客户端隔离，伙伴可能无法访问；
- 如果正在使用 VPN，建议关闭 VPN 后测试。

## 10. 知识库初始化与重建

系统启动时会检查 `data/index/` 中是否已有索引。如果索引不存在，且 `data/knowledge_base/` 中有文档，系统会自动构建索引。

内置知识文档位置：

```text
data/knowledge_base/
```

用户上传文档位置：

```text
data/knowledge_base/uploads/
```

索引文件位置：

```text
data/index/chunks.json
data/index/vectors.npy
data/index/metadata.json
data/index/faiss.index
```

可以在前端知识文档页面上传资料，也可以调用接口重建：

```text
POST http://127.0.0.1:8000/api/knowledge/rebuild
```

该接口需要登录 token。通常直接使用前端页面操作更方便。

## 12. 常见故障排查

### 12.1 端口 8000 被占用

`run.py` 默认会尝试关闭占用目标端口的旧进程。如果仍失败，可以换端口：

```powershell
D:\Anaconda\python.exe run.py --port 8010
```

访问：

```text
http://127.0.0.1:8010
```

### 12.2 MySQL 连接失败

检查：

- MySQL 服务是否启动；
- `.env` 中用户名和密码是否正确；
- `rag_job_qa` 数据库是否存在；
- MySQL 是否允许本机连接。

可重新执行：

```sql
CREATE DATABASE IF NOT EXISTS rag_job_qa
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

### 12.3 注册或登录失败

可能原因：

- MySQL 没有启动；
- 数据库账号密码配置错误；
- 用户名已存在；
- 密码少于 6 位；
- 表结构旧版本缺少字段。

解决方式：

```powershell
D:\Anaconda\python.exe -c "from src.rag_job_qa.config import Settings; from src.rag_job_qa.user_store import UserStore; UserStore(Settings.load()); print('用户表已检查')"
```

### 12.4 上传文档后问答检索不到

检查：

- 文档是否为空；
- 文件格式是否支持；
- 上传后是否完成索引重建；
- 问题是否和文档内容相关；
- 是否选择了错误的岗位分类过滤。

可以在知识文档页面查看文档是否入库，也可以打开：

```text
http://127.0.0.1:8000/api/knowledge/stats
```

### 12.5 大模型没有回答

检查：

- `.env` 中 `QWEN_API_KEY` 是否正确；
- `QWEN_BASE_URL` 是否是 OpenAI 兼容接口地址；
- 网络是否能访问模型服务；
- API Key 是否欠费、过期或没有权限。

### 12.6 Hugging Face 模型加载失败

这是可接受情况。系统会自动降级到本地 hashing embedding。

若要修复：

- 确认模型已下载到 `data/models/`；
- 确认 `.env` 中 `EMBEDDING_MODEL` 配置正确；
- 如果希望联网下载，把 `HF_LOCAL_FILES_ONLY=false`。

### 12.7 Neo4j 图谱为空

检查：

- Neo4j 是否启动；
- `.env` 中 `NEO4J_URI`、`NEO4J_USER`、`NEO4J_PASSWORD` 是否正确；
- 是否已经上传文档并重建知识库；
- Neo4j 版本是否支持向量索引。

即使 Neo4j 不可用，系统仍会显示本地兜底图谱。

### 12.8 校园网伙伴无法访问

检查：

- 是否使用 `--host 0.0.0.0` 启动；
- 伙伴访问的是否是你的 IPv4 地址；
- 防火墙是否放行；
- 两台电脑是否在同一网络；
- 校园网是否开启设备隔离。

## 13. 答辩前检查清单

- MySQL 已启动，能注册和登录；
- API Key 可用，问答能正常生成；
- 至少准备 2 到 3 份可上传的求职文档；
- 知识文档页面能看到文档列表；
- 知识问答能返回参考来源；
- 岗位助手能按岗位方向回答；
- 模拟面试能追问；
- 历史记录能查看；
- 知识图谱能展示，Neo4j 不可用时也有兜底图谱；
- 校园网演示时已提前测试伙伴访问地址。
